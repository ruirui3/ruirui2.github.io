⚙️ How Default JSON Loading Works
On every page load, the site attempts to fetch portfolio.json from the same directory as this HTML file on GitHub. If found, it overrides localStorage — so pushing a new portfolio.json to GitHub instantly updates the site on any device, even after a cache clear. LocalStorage only acts as a fallback when the file can't be fetched (e.g. running locally).
Workflow:
1. Edit your data here in the admin panel
2. Click Export → Download portfolio.json
3. Commit & push portfolio.json to your GitHub repo root
4. GitHub Pages serves the new file → all visitors see the update immediately

📤 Export portfolio.json

Download portfolio.json and commit it to your GitHub repo root alongside index.html. On every page load, the site fetches this file from GitHub and uses it as the source of truth — overriding any cached data.

🚀 Deploy Steps
  1. Download portfolio.json above
  2. In your GitHub repo, click Add file → Upload files
  3. Upload portfolio.json (alongside index.html)
  4. Click Commit changes
  5. GitHub Pages will serve the new data in ~30 seconds
  6. Any device loading your site will get the fresh data
